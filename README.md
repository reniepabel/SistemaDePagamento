# SistemaDePagamento
PITCH – SmartPay System (Sistema de Folha de Pagamento)
________________________________________
Apresentação
Olá, meu nome é Reniê Pereira Abel. Juntamente com meu parceiro de equipe, Allan Sóstenes, sou estudante de Análise e Desenvolvimento de Sistemas, e hoje vou apresentar o SmartPay System (SPS), o sistema de folha de pagamento, que foi desenvolvido em Java.
________________________________________
1. Problema ou Oportunidade
Muitas empresas, especialmente as pequenas, realizam o controle de folha de pagamento de forma manual, o que, por mais organizado que seja o processo, abre margem para erros no cálculo de salários, falta de padronização e gasto desnecessário de tempo.
Além disso, diferentes tipos de funcionários — como comissionados ou de produção — possuem regras específicas que tornam esse processo ainda mais suscetível a falhas.
________________________________________
2. Proposta de Valor
O SPS foi desenvolvido para resolver esse problema. Ele automatiza o cálculo de salários de forma simples, padronizada e confiável, garantindo que cada funcionário receba seu salário com base no seu tipo de vínculo, reduzindo erros humanos e tornando o processo mais eficiente e organizado.
________________________________________
3. Como funciona
O SPS foi desenvolvido com uma tecnologia confiável e amplamente utilizada no mercado, garantindo estabilidade, segurança e bom desempenho (ele utiliza linguagem Java e conceitos fundamentais de Programação Orientada a Objetos).
Ele possui:
•	Um menu interativo para navegação; 
•	Cadastro de funcionários com validação de dados; 
•	Regras de decisão automatizadas que permitem ao sistema identificar o tipo de funcionário e calcular o salário de forma correta e repetitiva para vários registros” (uso de estruturas como if, switch e loops); 
•	Três categorias de funcionários: 
o	Padrão (salário fixo), 
o	Comissionado (base + comissão sobre vendas), 
o	Produção (base + bônus por quantidade produzida); 
•	Cadastrar e armazenamento de múltiplos funcionários de forma organizada (armazenamento em lista dinâmica - ArrayList); 
•	E métodos organizados para cálculo e exibição dos resultados. 
A estrutura do SPS foi pensada para ser simples e eficiente, garantindo facilidade de uso e manutenção. Assim, se as regras de negócio forem alteradas, a mudança no código será rápido, simples e pontual (alterará somente um fração específica).
________________________________________
4. Plano de Uso
Esse sistema pode ser utilizado como base para pequenas empresas ou como protótipo educacional.
Ele pode evoluir facilmente para:
•	Integração com banco de dados; 
•	Uso de interface gráfica; 
•	Geração de relatórios automatizados; 
•	E até sistemas web ou mobile.”
________________________________________
5. Análise (Diferencial)
O diferencial do SPS não está na complexidade, mas na simplicidade, clareza e organização da solução.
Enquanto muitos controles são feitos manualmente ou em planilhas, esse sistema:
•	Centraliza informações; 
•	Automatiza cálculos; 
•	E reduz significativamente erros operacionais. 
________________________________________
Considerações Finais
Em resumo, este sistema é aplicável em cenário real de folha de pagamento, resolvendo imediatamente o processo administrativo de pagamento de funcionário de forma mais seguro e otimizada.
